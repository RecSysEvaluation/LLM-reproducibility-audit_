Source code

The review of the papers shows that a link to the source code is provided in 49 cases; two of these links point to empty repositories, and one points to an expired repository. However, most authors provide links to source code repositories, indicating that sharing code links has become mainstream practice.

Figure 2 summarizes the types of code provided across 46 scanned repositories. All repositories include code for the proposed models; however, three provide partial implementations. Since this code comes with instructions to train and test the model, it enables reproducing the results of the main technical contribution of the paper. For baseline models, source code is available in 12 repositories, three of which provide only partial code. Because our observations regarding baseline-code availability are consistent with a recent study [9], we restrict our analysis to collect artifacts required for the LLM-based proposed models. Other types of source code necessary for reproducibility include scripts for hyperparameter configuration and tuning, as well as data processing pipelines (e.g., preprocessing, and train/validation/test split generation). Hyperparameter tuning was applicable in 58 papers, of which five shared complete scripts, three provided partial scripts, and 11 fixed hyperparameter values without any justification. For data processing, the situation shows improvement, as about half of the papers provided preprocessing scripts.
